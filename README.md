# 积存金复盘台

积存金复盘台是一个面向黄金积存金交易截图整理的本地优先网页工具。项目聚焦“导入截图、OCR 识别、人工复核、结果汇总、历史批次管理”这一条闭环工作流，目标是将交易截图转换为可复盘、可导出、可继续维护的结构化批次数据。

## 项目简介

适用范围：

- 招商银行黄金账户历史交易截图
- 包含 `委托买入`、`委托卖出`、`已撤单`、`过期失效`、转换类条目的截图整理

非目标：

- 通用文档 OCR 平台
- 云端存储与自动同步
- 通用文档结构识别系统

当前支持：

- 将截图批量导入当前整理台
- OCR 识别后在明细表中人工复核与修正
- 将一个历史批次直接打开到当前整理台
- 将多个历史批次依次加入同一个当前整理台继续整理
- 在结果总览中查看实时市价与收益测算

## 运行与兼容性

### OCR 运行策略

项目采用双链路识别架构：

1. 本地 Python OCR 服务
2. 浏览器 OCR 回退链路

运行时行为：

- 前端优先探测本地 OCR 服务 `http://127.0.0.1:8765/health`
- 本地服务可用时，优先调用 Python + PaddleOCR
- 本地服务不可用时，自动回退到浏览器 OCR
- Python 返回结构化结果后，前端仍会通过共享解析器再次补跑解析，以统一字段口径并选择更合理的结果

### 浏览器兼容性

macOS 启动脚本当前行为：

- 如果检测到 Chrome，则优先使用 Chrome 打开
- 如果未检测到 Chrome，则自动回退到系统默认浏览器

因此，Chrome 不是运行前提；未安装 Chrome 时，项目仍然可以正常打开和使用。

推荐使用 Chromium 系浏览器进行导出测试，原因是这类浏览器通常对前台保存面板支持更完整，在以下场景中体验更稳定：

- 选择保存路径
- 修改导出文件名
- 保证保存面板以前台方式出现

在部分不支持相关能力的浏览器中，导出可能退化为直接下载到默认下载目录。

### 实时金价说明

当前版本的实时金价模块先使用 `Gold API` 作为公开参考价源。

适用定位：

- 用于结果总览中的实时参考价展示
- 用于本地“买入加点 / 卖出加点”换算
- 用于浮动收益和收益率的实时测算

边界说明：

- 该价格不是招商银行官方买卖价
- 该价格不是交易所官方前端成交接口
- 如需贴近实际招行价格，应结合本地加点参数一起使用

## 数据存储与隐私

### 浏览器本地数据

历史批次默认保存在当前浏览器的本地 `IndexedDB` 中，不写入仓库目录。

常见路径：

- Chromium 系浏览器：`~/Library/Application Support/<浏览器>/<Profile>/IndexedDB/`
- Safari / WebKit：`~/Library/Containers/com.apple.Safari/Data/Library/WebKit/WebsiteData/` 或 `~/Library/WebKit/.../WebsiteData/`

### 当前数据保留策略

当前版本将浏览器内历史批次视为本地临时工作数据：

- 关闭最后一个页面时会触发离开提醒
- 下次重新打开应用时，会清理上一轮浏览器内历史批次

如果需要长期保留，请在离开前手动导出 JSON。

### Git 与 GitHub 风险说明

浏览器本地 `IndexedDB` 数据不会被 git 自动跟踪，也不会自动推送到 GitHub。

需要注意的仅有手动导出的 JSON 文件：

- 若保存到仓库目录外，不会影响 git
- 若保存到仓库目录内，并手动执行 `git add` / `git commit`，则可能被纳入版本控制

## 导出说明

当前支持两类导出：

- 单个历史批次导出
- 全部历史批次导出

导出成功的前提：

- 页面通过 `http://127.0.0.1:4173` 打开
- 本地启动脚本仍在运行

验收标准：

- 触发导出后出现保存面板或下载行为
- 导出的 JSON 文件可正常保存到本地
- 再次导入该 JSON 时，批次可被识别

如果导出行为异常，请按以下顺序检查：

1. 执行 `npm run status`，确认本地实例正在运行
2. 确认当前页面地址为 `http://127.0.0.1:4173`
3. 确认当前页面不是旧页面缓存
4. 若浏览器不支持前台保存面板，导出可能退化为默认下载行为

## FAQ

### 1. 页面无法打开，应该先检查什么？

优先检查以下几项：

1. 执行 `npm run status`
2. 确认本地实例是否正在运行
3. 确认访问地址是否为 `http://127.0.0.1:4173`

常见原因：

- 本地服务未运行
- 当前页面是旧页面
- 端口已被现有实例占用

处理建议：

- 若显示未运行，重新执行 `npm run start`
- 若端口已被占用，启动脚本会优先尝试复用已有实例
- 若页面是旧页面，关闭标签页后重新打开应用

### 2. 关闭浏览器后，为什么历史批次不见了？

这是当前版本的预期行为，不是异常。

当前版本将浏览器内历史批次视为本地临时工作数据：

- 关闭最后一个页面时会触发离开提醒
- 下次重新打开应用时，会清理上一轮浏览器内历史批次

如果需要长期保留，请在离开前手动导出 JSON。

### 3. 导出按钮点击后没有明显反应，通常是什么原因？

常见原因：

- 保存面板被其他窗口遮挡
- 浏览器不支持前台保存面板，已退化为普通下载
- 页面已与本地服务断开

处理建议：

1. 执行 `npm run status`
2. 确认页面地址为 `http://127.0.0.1:4173`
3. 确认当前页面不是旧页面缓存
4. 建议优先使用 Chromium 系浏览器测试导出链路

### 4. 为什么导出文件会直接进入“下载”目录，而不是弹出保存面板？

这通常与当前浏览器能力有关。

说明：

- 支持前台保存面板的浏览器会优先弹出保存对话框
- 不支持相关能力的浏览器，导出可能退化为默认下载行为

处理建议：

- 优先使用 Chromium 系浏览器
- 确认页面是通过本地启动器或 `npm run start` 打开的新页面

### 5. 未安装 Python OCR 依赖时，项目还能用吗？

可以。

行为说明：

- 项目仍可正常启动
- OCR 会自动回退到浏览器链路
- 识别速度和稳定性通常不如本地 Python OCR

如果希望启用本地 OCR，请参考前文“安装本地 OCR 依赖”。

### 6. 为什么实时买入价和实时卖出价可能相同？

当前版本的实时金价模块先使用 `Gold API` 作为公开参考价源，本身只提供一条市场参考价。

因此：

- 当“买入加点”和“卖出加点”都为 `0` 时
- 实时买入价和实时卖出价会显示为相同数值

如果需要贴近实际招行价格，请结合本地“买入加点 / 卖出加点”一起使用。

### 7. 浏览器本地数据会不会被自动提交到 GitHub？

不会。

说明：

- 浏览器本地 `IndexedDB` 数据不会被 git 自动跟踪
- 也不会自动推送到 GitHub

真正需要注意的是你手动导出的 JSON 文件：

- 若保存到仓库目录外，不会影响 git
- 若保存到仓库目录内，并手动执行 `git add` / `git commit`，则可能被纳入版本控制
